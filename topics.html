section 01 · cognition

学习与认知

TOPIC 01
专注模式 vs 发散模式（灵感与孵化）
Focused vs Diffuse Mode · Incubation
已验证 · 点击播放 Learning how to learn · Barbara Oakley（TEDx）

对应 S31 主题："灵感不是等来的，是高压后留白逼出来的"。

  • 大脑有两种模式：专注（紧盯问题、走熟悉神经通路）与发散（放松走神、远距离连点）。
  • 洗澡、散步、睡前的灵光，正是发散模式在做联想——但前提是先有专注投入。
  • 方法：先猛攻 → 再刻意休息 → 让潜意识"孵化"，回来常有突破。
DMN = Default Mode Network（默认模式网络，休息时活跃的脑网络）
TOPIC 02
间隔重复 & 主动回忆
Spaced Repetition · Active Recall
已验证 · 点击播放 How to use Anki like a PRO — Justin Sung

对应 S31 主题：单词卡"把记忆钉得深"，复习节奏交给算法。

  • 记忆会沿"遗忘曲线"衰减；在快忘没忘时复习，记得最牢。
  • 主动回忆（盖住答案先自己想）比反复重读有效得多。
  • Anki 用 SRS 算法自动安排复习间隔：会的拉长、不会的缩短。
SRS = Spaced Repetition System（间隔重复系统）　FSRS = Free Spaced Repetition Scheduler

section 02 · engagement

教育与粘性设计

TOPIC 03
抽卡 / 变比强化心理
Gacha · Variable-Ratio Reinforcement
▶ YouTube 检索 看抽卡/盲盒为何让人停不下来的行为心理学

对应 S31 主题：单词卡的稀有度、盲盒、集齐兑换。

  • 变比强化（Skinner）：奖励在"不确定次数后"才出现，最容易上头。
  • 教育场景要克制——把"不确定奖励"绑在学习成果上，而非花钱抽卡。
  • 结合收藏完形（差一张就集齐）放大动力，但守住学习目标这条底线。
SSR = Super Special Rare（超稀有）　RNG = Random Number Generator（随机数）　LTV = Lifetime Value（客户终身价值）

section 03 · ai & architecture

AI 与架构

TOPIC 04
JEPA 与世界模型（Yann LeCun）
Joint-Embedding Predictive Architecture
已验证 · 点击播放 Yann LeCun · JEPA / World Models（Harvard CMSA）

S31 核心："先抓本质，再忽略噪声"——这就是它的源头。

  • 别在像素/原始空间重建一切，改在抽象表征空间预测，主动丢噪声。
  • 目标驱动 AI：用世界模型规划行动，而不是逐词生成。
  • 注意：原笔记把这套思想误标成了 GEPA，二者不同（见下条）。
JEPA = Joint-Embedding Predictive Architecture（联合嵌入预测架构）　LLM = Large Language Model

TOPIC 05
GEPA 反思式提示词进化
Genetic-Pareto Reflective Prompt Evolution
已验证 · 点击播放 Matei Zaharia · GEPA + DSPy（Berkeley RDI）

易混淆：GEPA ≠ JEPA。它是提示词优化器，不是模型架构。

  • 让模型反思自己的运行轨迹，诊断问题、提出更好的提示词。
  • 在"帕累托前沿"保留多条优解再重组，像进化算法。
  • 论文称比强化学习(GRPO)用更少尝试就能更好。
GEPA = Genetic-Pareto（遗传-帕累托）　RL = Reinforcement Learning（强化学习）　GRPO = Group Relative Policy Optimization
TOPIC 06
上下文工程
Context Engineering
已验证 · 点击播放 Context vs Prompt Engineering（IBM Technology）

对应 S31 主题：BNY 的 context.md、执行手册、云端记忆。

  • 不再只打磨一句提示，而是系统管理"模型每步该看到什么"。
  • 把上下文窗口当稀缺资源：检索、记忆、压缩、工具一起设计。
  • 解决 LLM"开新窗口就失忆"的根本痛点。
RAG = Retrieval-Augmented Generation（检索增强生成）　MCP = Model Context Protocol　KV cache = Key-Value 缓存
TOPIC 07
知识蒸馏
Knowledge Distillation
已验证 · 点击播放 Knowledge Distillation: How LLMs train each other（Julia Turc）

对应 S31 主题：Nuwa 的"蒸馏形/法/神"。

  • 严格义：用大"教师模型"训练小"学生模型"，省算力保性能。
  • S31 讲的多是提示层的"风格/方法迁移"，与训练级蒸馏不同。
  • 分清你要复制的是外观(形)、方法(法)还是视角(神)。
KD = Knowledge Distillation（知识蒸馏）　教师=Teacher，学生=Student

section 04 · engineering & tools

工程与工具

TOPIC 08
PARA 知识组织法
Projects · Areas · Resources · Archives
已验证 · 点击播放 Tiago Forte — The PARA Method

对应 S31 主题："PARA 是骨架"，知识库的物理结构。

  • 一切数字信息归入四类：项目 / 领域 / 资源 / 归档。
  • 精髓：按"可行动程度"分类，而不是按学科主题。
  • 项目有截止线，领域要长期维护；做完就从项目移到归档。
PARA = Projects（项目）/ Areas（领域）/ Resources（资源）/ Archives（归档）
TOPIC 09
JSON Schema 机读规则
JSON Schema · Validation
已验证 · 点击播放 What is JSON Schema（Automation Step by Step）

对应 S31 主题："Schema 是给机器读、给系统校验的规则层"。

  • 用 JSON 写的结构契约：规定字段、类型、取值范围。
  • 能被程序自动校验，不合规直接打回——像表单必填校验。
  • 只保证"结构对"，不保证"内容对"，仍需业务规则兜底。
JSON = JavaScript Object Notation　Schema = 模式/规格
TOPIC 10
有限状态机 FSM
Finite State Machine
已验证 · 点击播放 Finite State Machine Explained（ALL ABOUT ELECTRONICS）

对应 S31 主题：知识单元"知道自己在哪、下一步去哪"。

  • 系统任一时刻只处于有限个明确状态之一，靠事件合法转移。
  • 像快递物流：已揽收→运输中→派送中→已签收，不能乱跳。
  • 把固定流水线升级成可分叉、可回退、可审计的流程。
FSM = Finite State Machine（有限状态机）　DFA = Deterministic Finite Automaton
TOPIC 11
编译器中间表示 IR
Intermediate Representation
▶ YouTube 检索 看源码如何先转成"中间形态"再编译

对应 S31 主题：K-IR 缓冲层的灵感来源。

  • 源码不直接变机器码，中间先转成"便于分析优化"的形态。
  • 给上下游一个稳定统一的交换格式，减少传输损耗。
  • K-IR = 把这套思路搬到"知识"上：原料→结构化单元→成品。
IR = Intermediate Representation（中间表示）　K-IR = Knowledge Intermediate Representation
TOPIC 12
Git & GitHub 入门
Version Control · Clone · Push · PR
已验证 · 点击播放 Git & GitHub Crash Course — freeCodeCamp

对应 S31 主题："把源代码资产纳入自己的 AI 工作流"。

  • Git 是版本控制系统，记录每次代码改动、可随时回退。
  • GitHub 是放仓库的远端：clone 拉下来、push 推上去、PR 协作。
  • 这是"拉项目交给 AI 解构"这条能力链的第一步。
VCS = Version Control System（版本控制系统）　PR = Pull Request（合并请求）　repo = repository（仓库）
TOPIC 13
Obsidian 入门
本地 Markdown 知识库
已验证 · 点击播放 Obsidian 从0到1完整攻略（含同步+打通AI）· 中文

对应 S31 主题：本地 md 沉淀、双向链接、与 AI 配合。

  • 笔记以本地 Markdown 文件存放，数据归你自己、AI 可直接读。
  • 靠双向链接把笔记织成网络，而非死板的文件夹层级。
  • 强在文档沉淀，但对代码项目可视化支持有限（那交给 IDE）。
MD = Markdown（轻量标记语言）　vault = 库（Obsidian 的笔记仓）
TOPIC 14
Claude Code 终端 AI 代理
Agentic Coding in the Terminal
▶ YouTube 检索 看终端里的 AI 如何读改项目、跑命令

对应 S31 主题："下一步：把 Claude Code 用起来"。

  • 在终端里运行的 AI 编码代理：读代码、改文件、跑命令、用 Git。
  • 不是自动补全，而是项目级"自主执行"——给目标，它干活。
  • 正好承接你"拉项目 → 交 AI 解构 → 落地"的工作流。
CLI = Command-Line Interface（命令行界面）　IDE = Integrated Development Environment（集成开发环境）
